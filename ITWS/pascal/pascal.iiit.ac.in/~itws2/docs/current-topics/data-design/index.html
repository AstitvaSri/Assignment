1 What is a data type?

A data type specifies a category of data that share similar structural or interface properties. As a programmer, you will either work with built-in data types like number?, symbol?, etc., or you may need to define your own data types. Also the system may sometimes help you with some of the definitions needed when defining a new data type. There could be more complex data types such as vectors, functions, etc.

2 Key obligations when designing data types

When designing a data type it is necessary to address the following questions:

Construction
How is an instance of the data type constructed? Constructing Some data may be available as literals, e.g., strings, symbols etc. However, such data types could also have constructors. Others may be constructed by functions called constructors.
Type Identification
How is an instance of the data type identified? This is done using using functions called type predicates. These functions take a value and query the membership of that value in the data type. A data type is identified with its type predicate.
Extraction
How are its components extracted? This is done using extraction functions (or getters) if the data is compound, like a list.
Modification (if any)
How are its components modified? Modifying components of the data using setter functions if the data is mutable (like vectors).
Equality
When are two data instances equal? Functions to comparing data for structural equality.

2.1 Examples

2.1.1 Natural numbers nat?

Constructor
Natural numbers are already defined.
Type predicate nat?
A natural is a number that is an integer greater than equal to 0.
(define nat?  exact-nonnegative-integer?)
Equality
using =.

2.1.2 Lists list?

Constructors
'(), cons, list
Type predicates
list?.
Getters
first, rest, list-ref
Setters
none

2.1.3 Vectors vector?

Constructors
  • vector takes any number of arguments of and returns a vector of those argument.
  • make-vector takes two argument, a natural number n and a value v and returns a vector of size n each of whose element is v.
Type predicate
vector?
Getters
vector-ref
Setters
vector-set!

2.1.4 Strings

Constructors
literal (constant) strings (immutable). Also, string, a constructor analogous to vector in that it creates mutable strings.
(define s (string #\a #\b)) ; => "ab"
Type predicate
string?
Extractor/Getter
string-ref
(string-ref s 0) ; => #\a
Mutator/Setter
string-set!
(string-set! s 0 #\c)
(string-ref s 0) ; =#\c

3 Reference vs. structural equality in Racket

Two structures may be structural identical, yet referentially distinct as in they occupy different parts of the memory. Reference equality is checked with eq? in Racket. equal? checks structural equality.

3.1 Examples

Here are examples involving lists, vectors and strings.

3.1.1 Lists

Lists are constructed each time so they are referentially distinct. 

(eq? '(a b c) '(a b c)) ; => #f
(equal? '(a b c) '(a b c)) ; => #t

3.1.2 Vectors

Lists are constructed each time too. 

(eq? '#(a b c) '#(a b c)) ; => #f
(equal? '#(a b c) '#(a b c)) ; => #f

3.1.3 Strings

Two instances of literal strings are allocated the same memory and are therefor referentially identical. Two things that are referentially identical are of course structurally identical. 

(eq? "abc" "abc") ; => #t 
(equal? "abc" "abc") ; => #t

However, strings built with explicit constructors are allocated fresh memory: 

(eq? (string '#\a '#\b) "ab") ; => #f
(equal? (string '#\a '#\b) "ab") ; => #t
