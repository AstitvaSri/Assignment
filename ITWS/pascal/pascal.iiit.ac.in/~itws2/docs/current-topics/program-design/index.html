1 What is a program?

A Racket program is a collection of data and function definitions interspersed with testable assertions. The module makes available a subset of these definitions for use outside the module. Typically, a module is written in a file.

2 Preamble   boilerplate

The first line of each module file in Racket specifies the language of the module. 

#lang racket

This simply tells the Racket system that the program being written is in the racket language. The Racket system allows one to define other languages as well.

3 Function Design

A well designed function has the following parts:

Name
the name of the function
Purpose
A statement that describes what the program does, informally, but as precisely as possible.
Usage
Examples that show how to call the function and what to expect as a result, including any errors.
Signature
The types of the argument(s) and results
Definition
The definition of the function.
Test cases
Assertions that test your program
Other remarks
Other remarks about the function's behaviour, e.g., its performance, or boundary cases.

4 Functions on Fixed Data

4.1 Example: celsius->fahrenheit

4.1.1 Purpose

;;; celsius->fahrenheit

;;; Purpose
;;; =======

;;; This function takes temperature expressed 
;;; in celsius and returns
;;; equivalent temperature in fahrenheit.

4.1.2 Usage and Signature

;;; Usage
;;; =====
;;; (celsius->fahrenheit 0) 
;;;  => 32.0 (approximately)
;;; (celsius->fahrenheit 100) 
;;; ; => 212.0  (approximately)

;;; Signature
;;; =========
;;;  celsius->fahrenheit : temperature? -> temperature?
  1. Definitions
    ;;; Definitions
;;; ===========




;; celsius->fahrenheit : temperature? -> temperature?
(define celsius->fahrenheit
  (lambda (c)
    (+ (* c 1.8) 32)))

4.1.3 Testing

(require rackunit)
(require rackunit/text-ui)


(check-= 
 (celsius->fahrenheit 0.0) 
 32.0 "c0"))


;;; Approximate equality
;;; check-approx checks if two numbers are within 0.01 of each other.
;;; check-approx-= : number? number? -> void? 
(define check-approx-=
  (lambda (v1 v2)
    (check-= v1 v2 0.01)))

(check-approx-= 
 (celsius->fahrenheit 0.0)
 32.0 "c1"))

(check-approx-=
 (celsius->fahrenheit 100.0)
 212.0 "c2"))
