1 What is tagging?

A tag allows distinguishing data from another. Without the tag there could be confusion about which type of data is being referred to.

2 Example

  • Earlier, we chose temperature to be a real number. Recall the following definition we used:
(define temperature? real?)
(celsius->fahrenheit 0.0) 
;=> 32.0
  • So, you are holding the real number 32.0. Is it fahrenheit or celsius?
  • In engineering we use units and dimensions, which have a role similar to tags.

3 Choosing a tagged representation

  • celsius is a list of two elements.
  • the first is a tag 'celsius.
  • the second is a real number.
;;; celsius? : any/c -> boolean? 
(define celsius?
  (lambda (data)
    (and (list? data)
         (= 2 (length data))
         (eq? (first data)
              'celsius)
         (real? (second data)))))

4 Define a constructor

  • The constructor builds tagged data
;;; celsius : real? -> celsius?
(define celsius
  (lambda (temp)
    (list 'celsius temp)))

5 Extractor/Getter

;;; celsius->magnitude : celsius? -> real?
(define celsius->magnitude
  (lambda (cel)
    (second cel)))

6 Equality

;;; [any/c, any/c] -> boolean?
(define celsius-equal?
  (lambda (c1 c2)
    (and (celsius? c1)
         (celsius? c2)
         (equal? c1 c2))))

8 Define temperature as the union

;;; temperature? : any/c -> boolean?
(define temperature?
  (lambda (data)
    (or (celsius? data)
        (fahrenheit? data) 
)))

9 Using contracts

A type predicate is an example of a flat contract. There are built-in contract combinators that take contracts and return contracts. or/c takes two contracts and returns a contract. Very convenient. 

(require racket/contract)
(define temperature?
  (or/c celsius? fahrenheit?))
