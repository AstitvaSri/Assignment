1 Introduction

In this section, we understand the lexical structure of programs. We identify the different occurrences of variables (binding vs. reference) and whether the reference occurrences are bound to a binding occurrence or are free.

2 Binding and reference occurrences

In any expression, there are two types of occurrences of identifiers:

  • Binding occurrence
  • Reference (or usage) occurrence.

3 Binders

The following four constructs introduce binding occurrences of identifiers:

  • lambda
  • let
  • letrec
  • define

4 Binding occurrences: lambda

  • A lambda expression's formal parameters are binding occurrences.

    The occurrences in the body (unless they are binding occurrences) constitute reference occurrences. 

(lambda (x y)   ; these are binding occurrences of x and y
  (+ x y)       ; these are reference occurrences  x and y

5 Binding occurrences: let and letrec

A let expression contains a list of binds and a body, which is an expression. Each bind is a list of two things: an identifier (the bind identifier) and an expression (the bind expression). 

(+ x                   ; 1   reference occurrence of x
   (let ([x 5]         ; 2   binding occurrence of x as a bind identifier
         [y            ; 3   binding occurrence of y as a bind identifier
           (+ x 2)])   ; 4   reference occurence of x
    (+ x               ; 5   reference occurrence of x
       y)))            ; 6   reference occurrence of y

The rule for binding occurrences in a letrec are identical to that of a let.

6 Binding occurrence in a define

(define f                      ; 1  binding occurrence of f
  (lambda (n)                  ; 2  binding occurrence of n
    (if (= n 0)                ; 3  reference occurrence of n 
        1                      ; 4   
        (* n (f (sub1 n))))))  ; 5  reference occurrences of n and f

7 Occurrences are about structure, not behaviour

(+ x                   ; 1  reference occurrence of x
   (lambda (x)         ; 2  binding occurrence of x
     (let ([y 5])      ; 3  binding occurrence of y
       (+ x y z))))    ; 4  reference occurrences of
                       ;    x, y and z

Binding and reference occurrences have nothing to do with the program's behaviour, only its structure. ( For example, the above program, when run, gives an error.)

8 Free and Bound reference occurrences

A reference occurrence of an identifier in an expression may refer to a binding occurrence. In such a case the the reference occurrence is said to be bound to the binding occurrence.

In case there are multiple binding occurrences, the scoping rules decide which binding occurrence (if any) to which the reference occurrence is bound.

If the reference occurrence of an identifier in an expression refers to no binding occurrence, then that occurrence is free in that expression.

9 Identifying the binding occurrence

Given a program, how does one identify the the corresponding binding occurrence of a reference occurrence, if any?

Find the closest enclosing binder that has a binding occurrence of the identifier.

9.1 Example

(+ x                    ; 1  free reference occurrence of x
   (lambda (x)          ; 2  binding occurrence of x
     (let ([y  x])      ; 3  binding occurrence of y, 
                        ;    reference occurrence of x bound (to binding occurrence
                        ;    of x) on line 2 
        (+ y            ; 4  bound to the binding occurrence of x on line 2
           x            ; 5  bound to the binding occurrence of y on line 3
           z))))        ; 6  free reference occurrence of z

9.2 letrec vs. let

In letrec, the binding occurrence of a reference in a bind could be in the binds of the same letrec expression.

Here is an example of this phenomenon: 

(letrec ([x (lambda () y)]        ; 1. reference occurrence y is bound to 
                                  ;    binding occurrence of y in line 2

         [y (lambda () x)])       ; 2. reference occurrence x is bound to 
                                  ;    binding occurrence of y in line 2

   x)                             ; 3. reference occurrence of x is bound to 
                                  ;    binding occurrence of x in line 1

Compare this with a let expression:

The reference occurrences of x and y in lines 2 and 1 respectively are free: 

(let    ([x (lambda () y)]        ; 1. reference occurrence y is free


         [y (lambda () x)])       ; 2. reference occurrence x is free

   x)                             ; 3. reference occurrence of x is bound to 
                                  ;    binding occurrence of x in line 1

9.3 define

The rules of locating where a reference occurrence of a variable is bound in a define declaration is similar to that of a letrec. Think of a define conceptually (not syntactically) as a letrec with one bind and no body. Evaluating a define extends the top level environment (which we study next). 

(define f             ; 1 binding occurrence of f
   (lambda ()         ; 2
     f))              ; 3 reference occurrence of f bound to 
                      ;   the binding occurrence of f on line 1

10 Free and Bound identifiers

Free
An identifier is free in an expression if there is a free occurrence of the identifier in the expression
Bound
An identifier is bound in an expression if there is a bound reference occurrence of the identifier in the expression.

Note that an identifier may be free or bound (or both or neither) in an expression.
