1 Introduction

This is part 3 of the mini project. This assignment is about implementing APIs written in python using flask and sqlalchemy

2 Problems

In this assignment we will be creating the APIs which were already being used by you in the prevous assignment. The structure will involve the creation of 2 files, namely app.py and models.py. models.py will consist of 2 classes, one for student and the other for course. And app.py will consist of 5 functions corresponding to the following routes:-

  • /addStudent
  • /addCourse
  • /students
  • /courses
  • /enroll

The functionality of all these routes are as defined in the previous asignment. The only change is that in the /studuents route the student details will also consist of a list of courses that the student has taken

3 Assignment Instructions

3.2 Evaluation

  • You will be graded according to various other test cases as well as the quality of your solution.

3.3 Directory Structure

  • 1. Each student has access to a repository whose name is the roll-number of the student.
  • 2. In your GitLab repository, you must follow the directory structure as below:
     ---------
     <roll-no>
     |
     `--assignment-solutions
          |
          `- assignment-8
               |- app/
               |- run.py
               |- config.py
	       `...etc
     --------

3.4 Strict Adherence

If you do not adhere to this directory structure and naming, you will lose marks. The evaluation of this homework is automated. So, please make sure that you don't miss hyphens and upper case letters in file, directory and function names. No requests for regrading of your homework will be entertained if the directory structure or naming convention is not followed!
