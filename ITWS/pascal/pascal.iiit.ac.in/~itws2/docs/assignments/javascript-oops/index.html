1 Introduction

The aim of this assignment is learn how to develop the frontend for a web application using HTML and Javascript. This assignment is part 1 of 3 of the mini project.

2 Problems

Create the following

2.1 Student

2.1.1 Problem

  • Create the following object student as a safe object which has the following properties

    • roll constraint: [2011XXXX-2016XXXX] X => [0-9] 

               '20141065'

    • name constraint: only alphabets 

               'Bob'

    • year constraint: UGx x => [1-6] 

               'UG3'

    • courses constraint: [Course.id, Course.id, ...] 

               ['ICS101', 'ICS102', 'ICS103']

  • write a maker function makeStudent() for the object student that has arguments as roll, name, year and returns a safe object. 

           var s = makeStudent("20141065", "Bob", "UG3")
  • Add the following methods to the student object
    • getCourseIds() : returns the courses attribute
    • addCourse(id) : adds course id to the courses attribute
    • dropCourse(id) : removes course id from the courses attribute
    • findCourse(id) : checks if course id exists in the courses attribute and returns true or false

  var year = yr;

  return student;
}

3 Instructions

3.1 Jquery

You may use Jquery to process the forms and events. If you are doing so provide jquery inside index.html using a CDN or the downloaded script. 

   <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.1.1/jquery.min.js"></script>

3.2 Boilerplate

The boilerplate codes provided are not complete use them only as hints.

3.3 File template

  • Your final main.org file should tangle all the above defined javascript functions into main.js and all the html into index.html
  • Export your function definitions using module.exports

3.4 Evaluation

  • You will be graded according to various other test cases as well as the quality of your solution.
  • Manual evaluations will also be conducted.

3.5 Naming convention

  • 1. Use the same names of functions as mentioned in the questions.
  • 2. All the questions should be in the same file, and the file is to be called "main.org".
  • 3. "main.org" tangles out "main.js" and "index.html"

3.6 Directory Structure

  • 1. Each student has access to a repository whose name is the roll-number of the student.

  • 2. In your GitLab repository, you must follow the directory structure as below: 

         -------------------------
     <roll-no>
     |
     `----assignment-solutions
          |
          `--- assignment-6
               |--- main.org
     ------------------------

3.7 Strict Adherence

If you do not adhere to this directory structure and naming, you will lose marks. The evaluation of this homework is automated. So, please make sure that you don't miss hyphens and upper case letters in file, directory and function names. No requests for regrading of your homework will be entertained if the directory structure or naming convention is not followed!
