1 Introduction

This is part 2 of the mini project. This assignment is about implementing API calls using Javascript and integrating them with the frontend of the previous assignment.

2 Instuctions to run server

  • The files api.pyc and server.py must be in the same directory.
  • Use python 2.7 to run the API server. The server is hosted on 0.0.0.0:5000.
python server.py
  • Install Flask
pip install Flask
  • Install Flask-SQLAlchemy
pip install flask-sqlalchemy
  • Install Flask _CORS
pip install flask-cors
  • Installation using virtualenv
virtualenv venv (for ubuntu)
pyvenv venv (for fedora)
source venv/bin/activate
pip install Flask flask-sqlalchemy flask-cors

Implement the functions that make the API calls. The input should be taken from the forms defined in the previous assignment and the output should be displayed in the tables. The input and output of each API is defined as follows

3.2 To create a new student in the database

  • Route: "/addStudent"
  • Method: POST
  • Input: form
  • Form-Keys: 'roll', 'name', 'year'
  • Output-type: Text
  • Output: success, error

3.3 To create a new course in the database

  • Route: "/addCourse"
  • Method: POST
  • Input: form
  • Form-Keys: 'id', 'name'
  • Output-type: Text
  • Output: success, error

3.4 To fetch all students from the database

4 Assignment Instructions

4.1 File Template

  • Your final main.org file should tangle all the above defined javascript functions into main.js and all the html into index.html
  • Export your function definitions using module.exports
  • The functions should return the output of the API. This output must be displayed on the web page.

4.2 Evaluation

  • You will be graded according to various other test cases as well as the quality of your solution.

4.3 Naming convention

  • 1. Use the same names of functions as mentioned in the questions.
  • 2. All the questions should be in the same file, and the file is to be called "main.org".
  • 3. "main.org" tangles out "main.js" and "index.html"

4.4 Directory Structure

  • 1. Each student has access to a repository whose name is the roll-number of the student.

  • 2. In your GitLab repository, you must follow the directory structure as below: 

         ---------
     <roll-no>
     |
     `--assignment-solutions
          |
          `- assignment-7
               |- main.org
     --------

4.5 Strict Adherence

If you do not adhere to this directory structure and naming, you will lose marks. The evaluation of this homework is automated. So, please make sure that you don't miss hyphens and upper case letters in file, directory and function names. No requests for regrading of your homework will be entertained if the directory structure or naming convention is not followed!
